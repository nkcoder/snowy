## What is this

Snowy is a desktop PostgreSQL GUI client built with [Wails v2](https://wails.io/) — Go backend + React/TypeScript frontend bundled into a native app. The Go code exposes methods to the frontend via Wails bindings; the frontend calls them through auto-generated wrappers in `frontend/wailsjs/`.

## Commands

### Development
```bash
wails dev          # hot-reload dev mode (runs both Go and Vite)
```

### Build
```bash
wails build        # production binary with embedded frontend
```

### Frontend only (rarely needed standalone)
```bash
cd frontend && npm run dev      # Vite dev server alone
cd frontend && npm run build    # TypeScript check + Vite build
```

No test suite exists yet.

## Architecture

### Backend (Go)
- `main.go` — Wails bootstrap; embeds `frontend/dist` at compile time
- `app.go` — `App` struct with all methods bound to the frontend; orchestrates `ConfigManager` and `DbService`
- `config.go` — `ConfigManager` reads/writes `~/.snowy/config.json`; holds `Project` and `Datasource` structs (passwords stored plain text for now)
- `db_service.go` — `DbService` opens a fresh `pgx` connection per call (no pooling); implements schema/table/column introspection and query execution

### Frontend (React + TypeScript + Tailwind)
- `frontend/src/App.tsx` — single large component managing app-wide state; renders one of three views: `project-selector`, `datasources`, `workspace`
- `frontend/src/components/Sidebar.tsx` — lazy-loading tree (schemas → tables → columns) via Wails calls
- `frontend/src/components/QueryEditor.tsx` — SQL input + run button
- `frontend/src/components/ResultsTable.tsx` — tabular display of query results
- `frontend/wailsjs/go/main/App.js` — auto-generated JS bindings; **do not edit by hand** — regenerated by `wails dev`/`wails build`

### Wails binding contract
Any Go method on `App` that should be callable from the frontend must be exported and registered in `main.go`'s `Bind` slice. Wails generates the TypeScript/JS wrappers automatically. The frontend imports them from `../../wailsjs/go/main/App`.

## Key constraints
- Each DB call creates and closes its own `pgx.Conn` — no connection pool. Keep this in mind when adding features that need multiple sequential queries.
- Config is written atomically via `sync.RWMutex` but passwords are stored in plaintext in `~/.snowy/config.json`.
- `frontend/dist` must be built before `wails build` (the embed fails otherwise). `wails build` handles this automatically; manual `go build` does not.
